# DocMind 教材编辑器 - **完整端到端技术文档**

**版本**：v1.0（含阿里云解析链路）
**目标**：PDF → 阿里云 DocMind 解析 → Markdown 编辑 → 在线教材 HTML
**技术栈**：React + JavaScript + Vite + 阿里云 SDK（Node.js）

***

## 1. 完整技术流程图

```
PDF 文件
   ↓ (上传 OSS 或直传)
阿里云 DocMind 异步解析
   ↓ SubmitDocParserJob → QueryDocParserStatus → GetDocParserResult
前端解析器（JS）
   ↓ layouts → Markdown（带溯源）+ fragments 清单
React 编辑器
   ↓ 编辑 + 实时预览（教材主题）
HTML 导出器
   ↓ 静态 HTML（含 data-* 溯源属性）
```


***

## 2. 阿里云 DocMind 解析完整流程

### 2.1 API 序列（阿里云官方异步流程）

| 步骤 | API | 参数 | 返回 |
| :-- | :-- | :-- | :-- |
| 1 | `SubmitDocParserJob` | fileUrl (OSS URL), 输出类型 | jobId |
| 2 | `QueryDocParserStatus` | jobId | Status (running/success/failed) |
| 3 | `GetDocParserResult` | jobId, `layoutNum`, `layoutStepSize=100` | layouts 数组（分页） |

**关键点**：

- `layouts` 包含 `text`、`markdownContent`、`pageNum`、`index`、`uniqueId` 等。
- 长文档需多次调用 `GetDocParserResult`，用 `layoutNum` 分页。


### 2.2 Node.js SDK 调用（完整示例）

**文件**：`scripts/docmind-parser.js`

```js
// scripts/docmind-parser.js
// 完整：PDF URL → 阿里云解析 → 本地 Markdown + fragments

const Client = require('@alicloud/docmind-sdk'); // 阿里云官方 SDK
const fs = require('fs');
const { layoutsToMarkdown } = require('../src/parser/parseDocmindJson');

const client = new Client({
  accessKeyId: process.env.ALIYUN_ACCESS_KEY_ID,
  accessKeySecret: process.env.ALIYUN_ACCESS_KEY_SECRET,
  endpoint: 'docmind-api.cn-hangzhou.aliyuncs.com',
});

async function submitJob(fileUrl) {
  const req = new Client.SubmitDocParserJobRequest({
    fileUrl,  // OSS URL 或公网 PDF URL
    // 可选：outputMarkdown: true, extractImages: true 等
  });
  const resp = await client.submitDocParserJob(req);
  return resp.body.Data.id;
}

async function waitForJob(jobId, timeoutMs = 300000) { // 5分钟超时
  const start = Date.now();
  while (true) {
    const statusReq = new Client.QueryDocParserStatusRequest({ id: jobId });
    const statusResp = await client.queryDocParserStatus(statusReq);
    const status = statusResp.body.Data.status;
    
    console.log(`Job ${jobId} status: ${status}`);
    
    if (status === 'success') return true;
    if (status === 'failed') throw new Error('解析失败');
    if (Date.now() - start > timeoutMs) throw new Error('解析超时');
    
    await new Promise(r => setTimeout(r, 5000)); // 5秒轮询
  }
}

async function fetchAllLayouts(jobId) {
  const step = 100;
  let from = 0;
  let allLayouts = [];
  
  while (true) {
    const req = new Client.GetDocParserResultRequest({
      id: jobId,
      layoutNum: from,
      layoutStepSize: step,
    });
    const resp = await client.getDocParserResult(req);
    const layouts = resp.body.Data?.layouts || [];
    
    console.log(`获取第 ${from}-${from + layouts.length} 个 layouts`);
    
    if (!layouts.length) break;
    allLayouts = allLayouts.concat(layouts);
    if (layouts.length < step) break;
    from += step;
  }
  
  return allLayouts;
}

async function parsePdfToMarkdown(pdfUrl, outputDir = './docs') {
  try {
    // 1. 提交任务
    const jobId = await submitJob(pdfUrl);
    console.log(`任务提交成功，ID: ${jobId}`);
    
    // 2. 等待完成
    await waitForJob(jobId);
    console.log('解析完成');
    
    // 3. 分页拉取所有 layouts
    const layouts = await fetchAllLayouts(jobId);
    console.log(`总共 ${layouts.length} 个布局块`);
    
    // 4. 转换为 Markdown + 片段清单
    const { markdown, fragments } = layoutsToMarkdown(layouts, jobId);
    
    // 5. 保存文件
    const docId = jobId.replace(/[^a-zA-Z0-9]/g, '_');
    fs.mkdirSync(outputDir, { recursive: true });
    fs.writeFileSync(`${outputDir}/${docId}.md`, markdown, 'utf-8');
    fs.writeFileSync(`${outputDir}/${docId}-fragments.json`, JSON.stringify(fragments, null, 2), 'utf-8');
    
    console.log(`✅ 生成完成：${outputDir}/${docId}.md`);
    console.log(`   片段清单：${outputDir}/${docId}-fragments.json`);
    
    return { docId, markdown, fragments };
  } catch (error) {
    console.error('❌ 解析失败：', error.message);
    throw error;
  }
}

// CLI 使用：node scripts/docmind-parser.js https://oss.xxx.com/doc.pdf
if (require.main === module) {
  const [, , pdfUrl] = process.argv;
  if (!pdfUrl) {
    console.log('用法：node scripts/docmind-parser.js <PDF_URL>');
    process.exit(1);
  }
  
  parsePdfToMarkdown(pdfUrl);
}

module.exports = { parsePdfToMarkdown };
```

**环境变量**（`.env`）：

```
ALIYUN_ACCESS_KEY_ID=your_access_key
ALIYUN_ACCESS_KEY_SECRET=your_secret
```

**使用**：

```bash
node scripts/docmind-parser.js https://your-oss-pdf-url.pdf
```

**输出**：

```
docs/job123.md                 # 编辑用 Markdown
docs/job123-fragments.json     # 溯源清单
```

### 3.3 App.jsx（支持 2 种导入方式）

在前面 App.jsx 的基础上，增加「打开本地 .md 文件」：

```jsx
// 在 handleImportJson 后面新增
const handleImportMd = async (e) => {
  const file = e.target.files && e.target.files[0];
  if (!file) return;
  
  const text = await file.text();
  setValue(text);
  
  // 尝试解析出 fragments（如果有的话）
  const fragMatch = text.match(/<!--\s*#src:([^>]+)\s*-->/g);
  if (fragMatch) {
    // 简单解析，实际可更复杂
    console.log('检测到', fragMatch.length, '个溯源标记');
  }
};

// 在 JSX 中添加
<input type="file" accept=".md" onChange={handleImportMd} />
```

现在编辑器支持：

1. **导入阿里云 JSON** → 自动解析 → Markdown
2. **打开脚本生成的 .md** → 直接编辑

## 6. 常见问题与排查

### 6.1 阿里云解析失败

```
❌ 权限不足 → 检查 AccessKey 和 Region
❌ 超时 → 增加 waitForJob 的 timeoutMs
❌ layouts 为空 → 检查 PDF URL 是否公网可访问
```


### 6.2 JSON 结构不匹配

```
❌ layouts 路径不对 → 打印 resp.body.Data 确认结构
❌ markdownContent 为空 → 回退使用 text 字段
```


### 6.3 编辑器预览异常

```
❌ 主题 CSS 加载失败 → 检查 src/styles/ 路径
❌ HTML 导出乱码 → 确认 UTF-8 编码
```
